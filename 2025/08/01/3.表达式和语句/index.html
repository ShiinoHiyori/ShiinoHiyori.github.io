表达式和语句

运算符

​ 我们先来理清表达式和运算符的关系。

​ 表达式是由一个或多个操作数通过运算符按照一定的规则组合成的。而运算符——即具有运算功能的符号。根据使用运算符的对象个数，将运算符分为单目运算符、双目运算符和三目运算符。

单目运算符

​ 所谓单目运算符，即只对一个对象进行操作，作用通常是修改变量本身的状态或检查某种状态。我们所用到的单目运算符汇总如下：

运算符 名称 示例 说明
+ 正号运算 +a 返回 a 本身（几乎没意义）
- 负号运算 -a a 的相反数
++ 前/后自增 ++a, a++ a = a + 1
-- 前/后自减 --a, a-- a = a - 1
! 逻辑非 !a 如果 a 为真返回假，反之亦然
~ 按位取反 ~a 二进制位取反
* 解引用 *ptr 获取指针 ptr 指向的值
& 取地址 &a 获取变量 a 的内存地址
sizeof 类型大小 sizeof(a) 获取 a 所占字节数
typeid 类型信息 typeid(a) 返回类型信息（运行时类型识别）
new 动态分配 new int 分配动态内存
delete 释放内存 delete p 释放动态分配的内存
throw 抛出异常 throw 1 抛出异常
co_await 协程等待 co_await f() C++20 协程语法

注意！：自增加/减少存在前置和后置两种形式，其中前置形式，返回值是自身修改后的值，再参与其他运算；后置形式，先参加其他运算后在进行自身+1。

sizeof运算符的使用：功能是返回指定数据类型或者表达式值的数据类型在内存中占用的字节数。

  • sizeof(char)和sizeof(bool)占1个字节
  • sizeof(short)占2个字节
  • sizeof(int)和sizeof(float)占4个字节
  • sizeof(double)占8个字节

sizeof无法对void类型使用！

双目运算符

​ 双目运算符同理，需要两个操作数a和b才能运行，是最常用的一类运算符。双目运算符可细分为算术运算符、关系运算符、逻辑运算符、赋值与位运算符四种。

算术运算符

运算符 示例 含义
+ a + b 加法
- a - b 减法
* a * b 乘法
/ a / b 除法
% a % b 取余（整数）

注意！： 取余运算又称为模运算，机制是取整运算。

关系运算符

运算符 示例 含义
== a == b 是否相等
!= a != b 是否不等
> a > b 大于
< a < b 小于
>= a >= b 大于等于
<= a <= b 小于等于

注意！：

  • 请不要混淆=和==的概念！=代表赋值，==代表恒等！一个变量和一个常量判断是否相等时，可以写成1==a。

  • 请不要串接使用关系运算符！例如数学中的连续小于a<b<c，但是在C++语法中a<b<c是错误的！C++的语法中正确表达应该为a<b && a<c!

逻辑运算符

运算符 示例 含义
&& a && b 与（AND）
丨丨 a丨丨b 或（OR）

赋值与位运算符

运算符 含义 等价于表达式 示例
= 赋值 a=b a = b
+= 加后赋值 a = a + b a += b
-= 减后赋值 a = a - b a -= b
*= 乘后赋值 a = a * b a *= b
/= 除后赋值 a = a / b a /= b
%= 取余后赋值 a = a % b a %= b
<<= 左移后赋值 a = a << b a <<= 1
>>= 右移后赋值 a = a >> b a >>= 2
&= 按位与后赋值 a = a & b a &= b
丨= 按位或后赋值 `a=a b`
^= 按位异或后赋值 a = a ^ b a ^= b

隐式类型转换

​ 所谓的隐式转换，是指编译器在你没有明确要求的情况下，自动把一种类型的数据转换成另一种类型，以完成某些操作。例如：

​ 常见的隐式转换如下：

转换场景 示例 说明
基本类型自动提升 int -> double, char -> int 小范围类型自动提升为大范围类型
条件判断 if (x) 把 int 当成 bool 0 被当作 false，非零为 true
函数参数匹配 void func(double x); 传 int int 自动转换为 double
赋值操作 double x = 3; int 3 自动转换为 double 3.0

​ 注意！：隐式转换也会存在一定的风险！可能存在精度丢失或者逻辑错误的风险！

​ 为了避免不必要的风险，在隐式转换中，我们要遵循算术类型转换的规则。核心内容是确保转换兼容，将精度损失降到最小。

​ C++ 遇到两个不同类型的算术值进行运算，会把“较低等级的类型”转换为“更高等级的类型”。转换顺序如下（从低到高）：

转换等级 类型
1 bool → 转成 int
2 整型提升（char, short → int）
3 int, unsigned int
4 long, unsigned long
5 long long, unsigned long long
6 float
7 double
8 long double

强制类型转换

​ 所谓强制类型转换，就是程序员显式地告诉编译器，把一个类型的数据转换成另一个类型。

​ C++提供了四种类型转换运算符：

类型转换符 用途 是否安全 示例
static_cast 普通类型转换，如 int→double、指针类型之间的基本转换 ✅ double b = static_cast<double>(a);
const_cast 去掉变量的 const / volatile 修饰 ⚠️ 危险 char* p = const_cast<char*>(str);
dynamic_cast 用于类层次间的安全向下转型，必须有虚函数 ✅ Derived* d = dynamic_cast<Derived*>(base);
reinterpret_cast 强行转换指针类型或底层二进制表示 ❗ 非常危险 int* p = reinterpret_cast<int*>(0x12345678);

​ 虽然C++支持强制类型转换，但是我们要清楚的认识到**强制类型转换有时是危险的！**应该仔细考虑，如果可以使用其他方法可以达到目的，建议避免使用强制类型转换，不依赖强制类型转换也可以设计出好的程序。

循环语句

​ 学习完条件判断语句之后，我们将要重点学习循环语句，所谓循环语句，就是是让程序重复执行某段代码的语句，直到满足某个退出条件为止。循环语句的构成一般由四个部分构成，即初始体、判断条件、更新部分、循环体。

部分 作用
初始化 设置循环开始前的状态
判断条件 控制是否继续执行
更新部分 每次循环后的变量更新
循环体 每次执行的操作（代码块）

​ 我们将从最常见的while循环开始学习。

​ 所谓while，中文意思即当…..的时候，因此它的用法也正如它的中文意思一致：

​ 我们可以很轻易的发现while的流程即是先判断，再执行。因为我们已知的是循环的规则，但是不知道进入循环前的输入是否符合条件，所以while适用于条件不确定的循环。

​ 第二种则是for循环，它的用法如下：

​ 它的执行顺序是：

  • 先执行一次初始化语句

  • 执行一次条件表达式，若为假，则退出循环；若为真，则执行循环体语句一次，

  • 执行一次更新语句并且回到执行条件表达式，直到满足条件表达式为假。

​ 但是我们需要知道的是，对于for循环，并不需要写出所有的表达式，只要保留两个分号，这个循环就是合法的。例如我们可以省略初始化语句或者更新语句，一样可以执行循环。

​ 同理我们甚至可以省略所有的语句，但是需要注意的是它会变成一个死循环结构，必须手动break退出循环。

​ 第三种是do while循环，它是while语句的进阶用法。

​ 它的执行逻辑也很简单，do先执行一次代码，然后判断while条件，达到了条件之后再离开循环。我们不难发现，与while相比，它的特点就是至少要执行一次循环。

​ 讲完循环语句，我们也不得不提到与之对应的循环控制的内容。在C++中，循环流程控制的语句主要有三种，分别是break、continue和goto，我们将具体介绍它们的实际用法。

​ 首先是break语句，正如前文所提到的一样，break的作用就是在满足条件时，立即终止整个循环,跳出循环体，经常会搭配if来进行使用。

​ 我们可以看到，它的使用场景一般是在找到目标后用于退出循环，防止产生死循环。

​ 其次是continue语句，它的作用是跳过本次循环，进行下一次。或许难以理解，但是我们可以通过一个例子来进行学习。

​ 如上，我们的代码需求是在1-5之间不输出数字3，因此我们通过continue和if的搭配，实现对循环结果3的跳过。不难发现，它的使用场景通常是在不想执行某些特殊条件下的循环体代码。

​ 最后是goto语句，正如同它的中文意思一样，是去······地方。它的作用则是**跳转到程序中带有特定标签的位置。

​ 但是我们需要知道的是，一个程序中会出现各种各样的标签，例如start_1、start_2这种，使用goto语句难以避免的会导致结构混乱，倘若程序报错，进行维护是很困难的，因此在实际使用中很少会用到goto语句，仅作了解即可。

​ 除开上述3种之外，return也可以作为循环控制的语句，它常用于退出循环。与前面三种不同的是，它的位置一般处于函数体而非循环体之中，因为它的作用机制是立即终止整个函数的执行（包括循环），直接返回。

​ 理解了循环的主体和循环控制的用法之后，我们接下来要学习的就是循环嵌套，与上文所提到过的条件判断的循环嵌套所类似，即***在一个循环体内再写一个完整的循环结构。***利用循环嵌套，我们可以实现图形打印、表格输出、多级菜单和模拟结构行为等功能。例如我们可以使用循环嵌套来输出99乘法表。
